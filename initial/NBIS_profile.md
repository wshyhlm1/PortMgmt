## 模块2：竞争与护城河

### 2.1 竞争格局

Nebius处于快速增长的NeoCloud（AI云基础设施）市场，主要竞争对手包括：


| 竞争对手                   | 定位           | 与Nebius差异       |
| ---------------------- | ------------ | --------------- |
| CoreWeave              | 专注GPU云租赁     | 更聚焦，但规模小于Nebius |
| Lambda Labs            | GPU云服务商      | 规模较小            |
| IREN (Iris Energy)     | AI数据中心+比特币挖矿 | 多元化业务           |
| 超大规模云厂商(AWS/Azure/GCP) | 自有AI云服务      | 客户也可能成为竞争对手     |


### 2.2 Nebius的竞争优势

1. **全栈垂直整合能力**：从电力获取、数据中心设计建设到GPU部署和云平台软件，全流程自主掌控
2. **与NVIDIA深度绑定**：NVIDIA投资$20亿，形成战略合作，优先获取最新GPU（Blackwell等）
3. **大客户背书**：Meta（5年$30亿+）、Microsoft（多年协议）等顶级客户验证其可靠性
4. **Yandex技术遗产**：继承自Yandex的深厚工程能力和云平台建设经验
5. **全球布局**：美国、欧洲、中东多点布局，满足客户数据主权和延迟要求
6. **快速部署能力**：能在对需求、价格和利润有充分把握的情况下，迅速完成GPU部署

### 2.3 护城河评估

- **规模效应（强）**：快速扩张的GPU集群带来单位成本下降
- **客户锁定（中）**：大型合同期限延长，Q4平均合同期限增长50%
- **技术能力（中强）**：自研AI云平台、调度系统和推理优化技术
- **供应链关系（强）**：与NVIDIA等核心供应商的优先关系

**风险点**：

- 客户集中度高：Meta和Microsoft贡献大部分收入
- 超大规模客户可能最终自建产能（将工作负载收回内部）
- AI算力供需关系可能逆转，导致定价压力
- 资本密集度高，需要持续大规模融资

## 模块3：地缘政治与监管

### 3.1 地缘政治风险

- **俄罗斯关联风险**：公司前身为Yandex（俄罗斯科技巨头），虽已完全拆分，但市场仍关注其俄罗斯背景
- **创始人背景**：Arkady Volozh曾受欧盟制裁，后于2024年解除
- **数据中心布局**：在欧洲、美国和以色列建设，避免单一地区政治风险

### 3.2 关税影响

- 管理层在业绩会中表示关税议题正在持续关注，但认为短期内市场会自我调节恢复平衡
- GPU和数据中心设备可能受关税影响，但公司表示会及时通报进展

### 3.3 出口管制

- 需要关注美国对华AI芯片出口管制是否进一步升级
- 若管制扩大，可能影响GPU供应链或限制中国客户获取Nebius服务
- 公司主要客户为欧美企业，直接暴露相对有限

### 3.4 数据主权与隐私

- 作为云服务提供商，需遵守各地区数据保护法规（GDPR等）
- 多区域数据中心布局有助于满足数据本地化要求

## 模块7：AI Capex专项

### 7.1 Nebius作为AI Capex的核心支出方

Nebius是AI Capex周期中最纯粹的标的之一，其业务模式本质上是将大规模资本投入转化为AI算力租赁收入。

### 7.2 2026年AI Capex计划


| Capex项目 | 金额           | 说明               |
| ------- | ------------ | ---------------- |
| 总CAPEX  | $160亿 -$200亿 | 主要用于GPU部署和数据中心建设 |
| 电力获取    | <1%          | 相对低成本锁定未来电力资源    |
| 数据中心建设  | 重要部分         | 全球多站点同时推进        |
| GPU采购   | 最大部分         | Blackwell系列为主    |


### 7.3 Capex回报模型

- **Hopper系列**：预计约2-3年在毛利润层面实现收支平衡
- **Blackwell系列**：定位高端市场，回报周期有望更短
- **GPU定价**：基于每小时计算利润率定价，保持合理盈利水平
- **软件和服务收入**：利润率更高，可加速投资回报

### 7.4 资金来源与风险

- 约60%资金来自资产负债表、运营和已承诺资金
- 持续评估债务和股权融资选项
- 客户预付款提供额外资金来源
- 关键风险：若AI算力供需逆转，大规模CAPEX可能面临回报压力

## 模块8：AI模型发布专项

### 8.1 AI模型演进对Nebius的驱动

AI模型的快速演进（从大语言模型到多模态、Agentic AI、物理AI）持续推动对训练和推理算力的需求：

- **训练需求**：模型参数持续增长，需要更大规模GPU集群
- **推理需求**：模型部署后产生持续推理Token需求
- **Agentic AI**：需要支持复杂推理链和长上下文窗口
- **物理AI/机器人**：新兴需求，要求实时推理能力

### 8.2 Nebius对AI模型工作负载的支持


| AI模型阶段  | Nebius服务      | 说明                       |
| ------- | ------------- | ------------------------ |
| 预训练     | GPU集群租赁       | 大规模H100/H200/Blackwell集群 |
| 微调      | AI云平台         | 支持模型定制和领域适配              |
| 推理部署    | Token Factory | 推理即服务，优化成本和延迟            |
| Agent应用 | 全栈平台          | 支持Agent工作负载的完整生命周期       |


### 8.3 客户AI模型生态

- **Cursor**：AI编程助手，Agentic AI代表
- **World Labs**：空间智能领导者
- **各类AI实验室**：训练和运行前沿模型
- **企业客户**：将Nebius基础设施用于内部AI模型部署

### 8.4 前瞻：下一代AI模型影响

- 随着模型向多模态和Agentic演进，对算力的需求将从训练扩展到持续推理
- Nebius通过Token Factory和推理优化技术，正积极布局推理市场
- 收购Clarifai和Eigen AI强化了在推理系统优化方面的能力
