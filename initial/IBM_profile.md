### 1.2 公司标签

**软件、云厂商、AI基础设施、大模型、企业级AI、混合云、主机、咨询、IT服务**

### 1.3 核心业务卡位

IBM是全球最大的企业级IT解决方案提供商之一，定位于"企业AI操作系统"和混合云基础设施层，通过软件（Red Hat、watsonx）、咨询、基础设施（Z大型机、Power服务器）三位一体的模式，帮助大型企业将AI嵌入核心业务流程。

### 1.4 收入拆分

IBM运营四大业务板块：

- **Software（软件，~40%收入）**：包括混合云平台（Red Hat OpenShift）、自动化（Ansible、Terraform）、数据与AI（watsonx）、交易处理（主机软件）、安全软件。高价值年度经常性收入（ARR）基础已达227亿美元（同比+10%）。Red Hat收入增长14%，OpenShift收入增长超20%，ARR达17亿美元。
- **Consulting（咨询，~25%收入）**：提供业务转型、技术咨询、系统集成服务。GenAI咨询业务已超10亿美元，被Gartner评为GenAI咨询与实施服务新兴领导者。
- **Infrastructure（基础设施，~25%收入）**：包括IBM Z大型机、Power服务器、分布式基础设施及存储。z17大型机本季度增长67%，体现AI工作负载对主机的拉动。
- **Financing（融资，~2%收入）**：提供租赁、分期付款、贷款及短期营运资金融资。

## 模块2：竞争与护城河

### 2.1 市场地位

- **企业级混合云**：通过Red Hat OpenShift，IBM是全球最大的企业开源解决方案提供商之一，在受监管行业（金融、电信、政府）具有深厚根基。
- **大型机（Mainframe）**：IBM Z系列在全球关键任务工作负载（银行核心系统、航空订座、零售交易）中占据绝对主导地位， virtually no viable alternative。
- **企业AI咨询**：Gartner将IBM评为GenAI咨询与实施服务"新兴领导者"，在AI代理部署、企业流程再造方面具有先发优势。

### 2.2 核心护城河

1. **客户粘性与转换成本**：大型企业和政府机构对合规、审计追踪、混合云兼容性的要求极高，IBM在此领域拥有数十年积累的信任关系。
2. **主机生态锁定**：IBM Z客户超过70%持续扩展容量，主机软件栈（z/OS、Db2、CICS）与硬件深度绑定，形成极高的生态系统锁定效应。
3. **AI治理与可信度**：IBM强调"可治理的AI"（governable AI），其watsonx.governance、Granite Guardian等工具在监管严格的欧洲和美国受监管行业具有差异化优势。
4. **开源战略**：Red Hat、Ansible、Terraform等开源产品使IBM能够渗透多云环境，避免被单一云厂商锁定。

### 2.3 竞争格局

- **vs. 云巨头（AWS、Azure、GCP）**：IBM不直接在IaaS层竞争，而是定位在"企业AI中间件"层——模型编排、数据治理、自动化工作流。
- **vs. AI初创/OpenAI/Anthropic**：IBM Granite模型定位为"够用且可 permissively licensed"，在 air-gapped、受监管环境中具有部署优势，但不在benchmark竞赛中追求领先。
- **vs. 咨询公司（Accenture、McKinsey）**：IBM的差异化在于"技术+咨询"一体化，能够将AI战略直接落地到自家的软件和基础设施上。

## 模块3：地缘政治与监管

### 3.1 监管环境

- **AI监管**：IBM积极拥抱监管，认为"主权（sovereignty）是AI的核心要求而非可选项"。IBM Sovereign Core平台支持在严格地理边界内部署AI，满足欧洲GDPR及各国数据本地化要求。
- **美国联邦支出**：2025年上半年美国联邦支出受到一定限制，对IBM政府业务产生短期影响，但管理层认为不构成长期阻力。
- **并购监管**：管理层对当前并购环境表示乐观，认为"有意义的并购交易将在合理时间内获批"。HashiCorp和DataStax的整合进展顺利。

### 3.2 地缘政治风险

- **关税与贸易摩擦**：尽管地缘政治紧张局势促使部分客户采取谨慎态度，但IBM管理层表示这并非整体主要因素。
- **数据主权**：随着各国加强数据本地化立法（如欧洲AI法案、中国数据安全法），IBM的Sovereign Core和混合云战略具有结构性受益逻辑。
- **中国业务**：IBM在中国的传统硬件和咨询业务持续收缩，但非公司核心增长驱动力。

### 3.3 供应链安全

- IBM z17和Power11处理器为自研（Telum II、Power11），在高端芯片供应链安全方面具有一定自主性。
- 与AMD、Intel等合作伙伴的芯片供应关系稳定。

## 模块4：技术路线图

### 4.1 主机与处理器

- **z17（2025年发布）**：搭载全新Telum II处理器，每天可提供超450亿次AI推理操作，延迟仅毫秒级。Spyre加速器将于2025年Q4推出，支持watsonx Z代码助手和Z助手在z17上原生运行。
- **Power11（2025年7月推出）**：面向混合环境中关键任务、数据密集型工作负载，性能、弹性和可扩展性大幅提升。

### 4.2 企业AI平台（watsonx）

- **watsonx Orchestrate**：从预览版进入正式版（GA），进化为多代理时代的"代理控制平面"，支持150+预构建企业连接器（Salesforce、Workday、ServiceNow、SAP等），通过MCP/A2A协议集成第三方代理。内置可观测性仪表盘，满足受监管行业合规要求。
- **watsonx.data**：扩展实时上下文层（Context in watsonx.data），通过联邦数据治理使企业AI能够对业务数据进行可靠推理。集成Confluent实时数据流（Kafka/Flink）。
- **IBM Concert**：AI驱动的运营平台，从被动监控转向协调智能响应，覆盖应用、基础设施、网络的全栈关联分析。

### 4.3 Granite模型家族

- **Granite 4.1（2026年4月发布）**：包括小语言模型（SLM）、语音、视觉、嵌入、Guardian模型。训练于约15万亿token，上下文长度扩展至512K token。采用Apache 2.0许可证开源发布。
- **定位**：不追求benchmark领先，而是强调工具调用、指令遵循、可预测延迟和更低运营成本，适合企业生产环境部署。

### 4.4 量子计算

- 2025年Q2，IBM与理化学研究所在日本部署了IBM Quantum System Two，为美国以外的首次安装，表明IBM在全球量子计算领域的领导地位。

### 4.5 收购整合路线图

- **HashiCorp**：首个完整季度表现加速，Ansible+Terraform联合产品取得早期成功，推动自动化业务增长14%。
- **DataStax**：增加实时可扩展数据能力，支持基于AI的应用程序。
- **Confluent**：实时数据流成为AI代理协调的数据基础设施前提。

## 模块7：AI Capex专项

### 7.1 IBM的AI CAPEX策略

IBM不直接参与大规模AI数据中心建设（如微软、谷歌、亚马逊的千亿美元级GPU集群），而是采取**"轻资产+平台层"**策略：

- **基础设施层**：通过z17大型机（内置Telum II AI加速器）和Power11，将AI推理能力嵌入企业本地数据中心，无需外部GPU服务器即可在事务工作负载上直接运行AI。
- **平台层**：watsonx作为企业AI的"操作系统"，提供模型开发、数据治理、代理编排，支持客户在混合云/本地/边缘环境中部署AI。
- **生态层**：与AWS、Microsoft、Salesforce、Oracle等合作，将watsonx嵌入其云平台，实现"无处不在的AI"。

### 7.2 AI CAPEX与收入映射


| 投入方向                             | 具体动作                           | 商业成果                 |
| -------------------------------- | ------------------------------ | -------------------- |
| z17/Power11研发与产能                 | 16.17亿美元CAPEX中相当部分投入主机/服务器     | z17增长67%，带动软件和服务捆绑销售 |
| watsonx平台研发                      | 约83.20亿美元R&D支出（2025年）中显著比例投向AI | GenAI累计预订超75亿美元      |
| 并购（HashiCorp/DataStax/Confluent） | 数十亿美元级并购支出                     | 扩展自动化、实时数据、开源生态能力    |
| 合作伙伴生态                           | 与 hyperscaler 联合推广             | 降低客户获取成本，扩大市场覆盖      |


### 7.3 与HyperScaler CAPEX的对比

- HyperScaler（微软、谷歌、亚马逊）的CAPEX主要投向GPU/TPU集群和数据中心建设，属于"重资产"模式。
- IBM的CAPEX主要投向软件平台、主机处理器和并购整合，属于"轻资产、高毛利"模式，资本密集度远低于云厂商。

## 模块8：AI模型发布专项

### 8.1 Granite模型家族演进


| 版本              | 发布时间        | 关键特性                                                        |
| --------------- | ----------- | ----------------------------------------------------------- |
| Granite 3.x     | 2024-2025   | 企业级小模型，注重工具调用和代码生成                                          |
| **Granite 4.1** | **2026-04** | **上下文长度扩展至512K token；覆盖SLM、语音、视觉、嵌入、Guardian；Apache 2.0开源** |


### 8.2 Granite 4.1技术亮点

- **训练数据**：约15万亿token，多阶段训练：从 broad pre-training 逐步退火至高质量技术/科学/数学数据，专注指令遵循。
- **后训练**：多阶段强化学习（RL）流水线，每阶段针对不同能力（对话、事实准确性、数学推理），避免单阶段优化的权衡。
- **企业适用性**：不依赖长思维链，提供可预测延迟、稳定token使用量和更低运营成本，定位为"生产就绪"的企业工作负载选择。
- **Guardian 4.1**： harm detection 模型，扩展风险定义，为开发者评估模型输入输出提供更细粒度信号。

### 8.3 watsonx Orchestrate：代理编排层

- **定位**：从单代理工具进化为"多代理控制平面"，支持跨异构环境部署来自任何来源的代理，并实施一致策略执行和责任追溯。
- **核心功能**：
  - 监督代理（Supervisor Agent）：基于微调Granite推理模型，分解用户请求，路由至内部代理、第三方代理（MCP/A2A协议）、确定性技能或人工审核员。
  - 150+预构建企业连接器 + 700+ webMethods集成，覆盖Salesforce、Workday、ServiceNow、SAP等。
  - 可观测性仪表盘：暴露每次路由决策，满足金融行业等强监管要求。

### 8.4 与第三方模型的关系

IBM明确**不直接与OpenAI、Anthropic在基础模型benchmark上竞争**：

- Granite定位为"Linux之于macOS"——不需要赢得benchmark，而需要在 air-gapped、受监管、本地部署环境中可落地。
- 通过watsonx Orchestrate，IBM支持集成Claude、GPT、Llama等第三方模型，自身聚焦在"编排、治理、数据"层。

### 8.5 发布节奏展望

- 预计2026年下半年将持续迭代Granite模型，特别是在多模态（语音+视觉+文本）和企业垂直领域微调方面。
- watsonx Orchestrate的代理生态将从150个预构建代理扩展至更多行业特定场景。
